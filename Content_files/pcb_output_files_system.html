SECTION 01

Overview of PCB Output Files

What Are PCB Output Files?

PCB output files are the complete set of digital data packages exported from a CAD design tool (Cadence Allegro, Altium Designer, etc.) that define every physical and electrical aspect of a printed circuit board. These files are the authoritative instructions handed to fabrication houses, assembly contractors, and test engineers.

A PCB design that exists only in a CAD tool cannot be manufactured. Output files translate the design intent into machine-readable and human-readable instructions for every stage of production.

Why They Are Critical for Manufacturing

  • Fabrication houses rely on Gerber/ODB++ files to image copper layers, drill holes, and cut board outlines with CNC equipment.
  • Assembly houses rely on pick-and-place files and BOMs to load SMT machines and apply solder paste stencils.
  • Test engineers use IPC-D-356 netlists and test point reports to configure automated test equipment (ATE).
  • A single error in any output file can result in scrapped boards, costly re-spins, or field failures.

Three Categories of Output Files

SECTION 02

Fabrication Output Files (DFM – Core)

2.1 Gerber Files (RS-274X / X2)

Gerber is the universal photoplotter format used by virtually all PCB fabrication facilities. Each physical layer of the PCB is represented by a separate Gerber file. RS-274X (Extended Gerber) is the current standard; Gerber X2 adds structured metadata (layer type, polarity, etc.) embedded within the file itself.

Layer File Map

Aperture Definition & Polarity

📐 Aperture Definition (RS-274X)

In RS-274X, apertures (pad shapes, trace widths) are defined inline within the Gerber file itself using %ADD statements. This eliminates the need for a separate aperture file (.apt), which was required by the older RS-274D format.

  • Standard shapes: Circle (C), Rectangle (R), Oblong (O), Polygon (P)
  • Custom macros: Defined with %AM for non-standard pad geometries
  • Always verify aperture list matches CAM tool expectation — mismatched apertures produce wrong pad sizes at FAB.
⚠️ Polarity — Solder Mask & Planes
  • Positive polarity: Dark areas represent copper/material present. (Used for copper layers, silkscreen)
  • Negative polarity: Dark areas represent material removed. (Historically used for internal power planes)
  • Sending a negative-polarity plane Gerber to a FAB expecting positive polarity = complete copper/void inversion. All traces and pads are swapped with clearances — catastrophic.
  • Gerber X2 includes polarity metadata in the header to prevent this.

2.2 Drill Files (Excellon Format)

The NC Drill file instructs the CNC drill machine on hole size, location, and type. The Excellon format is the industry standard. One drill file typically covers all plated holes; a separate file covers non-plated holes.

📋 Fabrication Drawing — Mandatory Notes
  • Board dimensions: Length × Width ± tolerance (e.g., 120mm × 80mm ±0.15mm)
  • Surface finish: HASL / ENIG / ENEPIG / OSP — specify clearly
  • Controlled impedance: Specify layer, target Ω, tolerance (e.g., L1 50Ω ±10%, single-ended)
  • Copper weight: Inner layers often 0.5 oz for high-density designs
  • Min trace/space: e.g., 0.1mm/0.1mm (4mil/4mil)
  • Via type: Through-hole, blind, buried — note span and fill (open, tented, plugged)
  • UL/RoHS marking: Required for most consumer and industrial products

2.4 Board Outline (Profile)

The board outline defines the exact physical boundary of the PCB. It is drawn on a dedicated mechanical layer and exported as a separate Gerber layer or embedded in ODB++. All features must form a closed polygon with no gaps, overlaps, or duplicate lines.

  • Standard layer: GM1 (mechanical), GKO (keep-out/outline in Altium)
  • Line width: 0mm (zero-width) is acceptable for profile; FAB reads centerline of outline
  • Tolerances: Routed boards ±0.15mm; V-scored boards ±0.3mm
  • Cutouts/slots: Must be drawn on same layer with tool compensation note in fab drawing
  • Fiducial marks: 3× minimum on panel/board for SMT machine calibration
SECTION 03

Assembly Output Files (DFM – Critical)

3.1 Pick and Place File (Centroid File)

The pick-and-place (PnP) or centroid file instructs SMT placement machines on the exact location and orientation of every component. Errors in this file cause misplaced, rotated, or skipped components — some of which are only discovered during functional test or in the field.

3.2 Bill of Materials (BOM)

The BOM is the complete parts list required to procure and assemble the board. It must be the single source of truth for all components. Errors cause wrong parts ordered, assembly with incorrect components, and procurement delays.

3.3 Assembly Drawing

The assembly drawing is a human-readable document (typically PDF) that shows component placement, orientation marks (polarity, pin 1), and special assembly notes that SMT/THT operators need.

  • Component outline view: Top and bottom views with all reference designators labeled
  • Polarity marks: Cathode stripe on diodes, pin 1 indicator on ICs, positive terminal on capacitors
  • Special notes: Connectors facing direction, heat sink orientation, conformal coating keepout zones
  • Revision block: Drawing number, revision, date, engineer approval signature
  • Dimension reference: Overall board dimensions for quick sanity check

3.4 Solder Paste Layer (Stencil Design)

The solder paste Gerber (.GTP / .GBP) is used by the stencil manufacturer to laser-cut or electroform the SMT stencil. Correct aperture design is critical to achieving reliable solder joints per IPC-A-610.

⚠️ Aperture Reduction Rules (IPC-7525)
  • Standard rule: Aperture area ratio ≥ 0.66 (area of aperture ÷ area of aperture walls) for good paste release
  • Typical reduction: 10–20% area reduction per side for QFN/BGA pads
  • Fine-pitch components (<0.4mm pitch): Require home-plate or custom aperture shapes to prevent bridging
  • BGA: Round apertures at 85–90% of pad diameter, avoid solder balling
  • No paste on vias: Tented vias in paste layer cause solder wicking into the via
  • Stencil thickness: Typically 0.12mm (5mil) for mixed technology; may require step stencil for fine-pitch + large pads
SECTION 04

Test Output Files (DFT – Core)

4.1 Netlist File (IPC-D-356 Format)

The IPC-D-356 netlist is the standard format for communicating PCB connectivity to In-Circuit Test (ICT) systems, flying probe testers, and DFT (Design for Test) analysis tools. It provides a pin-to-pin connection table derived from the schematic netlist, correlated with physical pad locations.

📡 IPC-D-356A Record Types
  • Record 317: Through-hole pad — net name, ref des, pin number, XY coordinate, drill size, access side
  • Record 327: SMT pad — net name, ref des, pin number, XY coordinate, pad size, access side
  • Record 999: End of file delimiter

IPC-D-356A is the revision used by modern ATE. Legacy IPC-D-356 (without revision A) may not support certain extended features — verify with test team.

4.2 Test Point Report

The test point report lists all accessible test nodes — nets that have a physical probe point (test pad, via, or through-hole pin) reachable by ICT or flying probe. Test coverage is measured as the percentage of nets with at least one accessible test point.

ParameterIPC TargetNotes
Test point diameter Min 1.0mm (ICT), 0.8mm (flying probe) Smaller pads risk probe miss, solder mask damage
Clearance around test point Min 2.54mm (100mil) pitch for ICT grid Denser layouts require flying probe or bed-of-nails fixture
Net coverage target ≥80% of nets (aim for 95%+) Low coverage = undetected opens/shorts, field escapes
Power/GND access Mandatory At minimum 1 test point per power rail
Component side access Top preferred; bottom if no top TP Bottom-only access adds fixture complexity

4.3 Boundary Scan Files (BSDL)

Boundary Scan Description Language (BSDL) files describe the IEEE 1149.1 (JTAG) boundary scan architecture of each IC on the board. Used by boundary scan test tools (XJTAG, Goepel, JTAG Technologies) to test connectivity between JTAG-capable devices without physical probe access.

  • BSDL files are supplied by the IC manufacturer — not generated by the PCB CAD tool
  • Board-level BSDL (.bsd) may be generated by tools like XJTAG Board Designer
  • Applicable only when design contains JTAG-capable devices (FPGAs, DSPs, modern MCUs)
  • If no JTAG-capable devices are present: Not applicable — Data Not Available from SSOT
SECTION 05

Advanced Output Formats

5.1 ODB++ (Open Database++)

ODB++ is a complete manufacturing database developed by Valor (now Siemens EDA). Unlike individual Gerber/Excellon files, ODB++ is a single compressed archive (.tgz or directory) that contains all PCB manufacturing data — layers, drill, netlist, stackup, components — in a structured, relational database.

FeatureGerber + ExcellonODB++
Data format Multiple separate flat files Single unified database
Embedded netlist No (separate IPC-D-356) Yes, included natively
DFM analysis Limited — FAB must correlate files Full DFM possible from single file
Layer correlation errors Possible (human error) Eliminated — layers internally correlated
Assembly data Separate PnP + BOM files Included in database
Industry adoption Universal (all FABs accept) High (most major FABs; verify before sending)
File corruption risk Low per file; high across file set Checksum validation possible per archive

5.2 IPC-2581 (Unified Data Format)

IPC-2581 (also called DPMX — Design to Process Management Exchange) is an XML-based open standard developed by the IPC industry consortium to replace both Gerber and ODB++ as a vendor-neutral, fully structured PCB data format. Adoption is increasing rapidly, particularly among companies with multi-FAB sourcing strategies.

✅ IPC-2581 Key Advantages
  • Vendor-neutral: No proprietary license required (unlike ODB++)
  • Single file: All fabrication, assembly, and test data in one XML file
  • Bidirectional: FAB can send DFM feedback in the same format
  • BOM linkage: Component data directly linked to footprints — eliminates BOM/PnP mismatch
  • Version B (2014+): Adds rigid-flex support, improved assembly data
  • CAD support: Cadence Allegro 17.4+, Altium 23+, Zuken CR-8000
SECTION 06

File Generation in CAD Tools

Cadence Allegro PCB Designer

Artwork (Gerber) Generation

  1. Navigate to File → Export → Artwork
  2. In the Artwork Control Form, verify each layer is mapped to the correct film name
  3. Set format to Gerber RS-274X
  4. Configure plot origin (board datum), units, and decimal precision
  5. Enable Suppress unconnected pads if required by FAB
  6. Generate each layer individually or use Create Artwork batch mode
  7. Verify output: open generated .gbr files in a Gerber viewer (Gerbv, ViewMate)

Drill File Generation (NC Drill)

  1. Navigate to Manufacture → NC → NC Drill
  2. Select output format: Excellon
  3. Set units (metric recommended), coordinate format, zero suppression mode
  4. Generate separate files for PTH and NPTH
  5. Generate separate files for each blind/buried via span if applicable
  6. Cross-check drill chart: tool count and sizes must match FAB requirements

Altium Designer

Altium uses an Output Job File (.OutJob) to define and batch-generate all output files in a single, version-controlled configuration.

Setting Up the .OutJob File

  1. Go to File → New → Output Job File
  2. Under Fabrication Outputs, add: Gerber Files, NC Drill Files, ODB++ (optional), IPC-2581 (optional)
  3. Under Assembly Outputs, add: Pick and Place Files, Assembly Drawings (PDF)
  4. Under Report Outputs, add: Bill of Materials (BOM), Test Point Report
  5. Under Netlist Outputs, add: IPC-D-356A Netlist
  6. Configure each output: layer selection, format settings, output directory
  7. Run all outputs via Validate All then Generate Content
💡 Altium Gerber Layer Configuration (Critical Settings)
  • Plot Layer: Ensure all copper, mask, silkscreen, paste layers are checked — not just active layers
  • Mirror layers: Bottom layers must be mirrored correctly for FAB (verify with viewer)
  • Apertures: Use Embedded apertures (RS-274X) — never RS-274D
  • Resolution: Set to 4:5 or 5:6 decimal format — 2.5 is legacy and insufficient for modern designs
  • Include pad holes in solder mask: Ensure controlled — should NOT have solder mask on via pads if tenting not specified

SECTION 08

Validation & CAM Check

Design Rule Check (DRC) — In CAD Tool

DRC must be run and pass with zero critical violations before any output files are generated. DRC enforces the design against a rules file that contains FAB and assembly constraints.

  • Clearance rules: Trace-to-trace, trace-to-pad, pad-to-board-edge, via-to-via
  • Annular ring rules: Minimum copper ring remaining after drilling (typically ≥0.1mm)
  • Solder mask rules: Mask expansion, solder mask slivers, via tenting consistency
  • Silk rules: Text height, line width, no silk over bare copper (pads)
  • Impedance rules: Trace width on controlled-impedance layers within spec

CAM Verification Steps

🔍 Mandatory CAM Checks (Valor NPI / CAM350)
  • Layer alignment: Load all Gerber layers in CAM tool; verify copper, mask, silk, paste all align with board outline. Check drill overlay on copper — all pad centers must have a coincident drill hit.
  • Drill-to-copper alignment: Drill file overlaid on copper Gerbers — annular ring must meet spec on all layers for all drill hits.
  • Netlist comparison: Import IPC-D-356 netlist into CAM tool; run netlist-versus-Gerber compare — all nets must match with no opens or shorts detected.
  • Silkscreen cleanup: Verify no silkscreen features overlap solder mask openings.
  • Board outline closed: Confirm outline polygon is fully closed with no gaps, overlaps, or T-intersections.
  • Fiducial check: Minimum 3 global fiducials on panel; correct shape (1mm circle in 3mm clear area).

Naming Convention Rules

  • Format: PROJECTNAME_LAYER_TYPE_vMAJOR.MINOR.YYYYMMDD
  • Versioning: Increment minor version (v1.1, v1.2) for non-stackup changes; major version (v2.0) for stackup changes or re-spins
  • No spaces in filenames: Use underscores only — FAB CAM scripts break on spaces
  • Archive format: ZIP or TAR.GZ with checksum (MD5/SHA256) for integrity verification
  • Release tag: Append _REL for released packages; _PROTO for prototype-only releases
SECTION 10

Real-World Release Flow

Final DRC Check in CAD Tool
Run full DRC in Cadence Allegro or Altium. Achieve zero critical violations. Document results in DRC report. Lock the design — no further changes after DRC pass.
Generate Fabrication Files
Export all Gerber layers, NC drill files (PTH + NPTH), and fabrication drawing PDF. Verify file count matches expected layer count. Open in Gerber viewer immediately after generation.
Generate Assembly Files
Export pick-and-place (top and bottom), paste mask Gerbers, and assembly drawing PDF. Generate BOM from the same design revision. Cross-validate BOM ref des list against PnP ref des list.
Generate Test Files
Export IPC-D-356A netlist and test point report. Calculate test coverage percentage. If coverage <80%, flag to DFT engineer before release. Export BSDL if boundary scan is in scope.
CAM Validation (Internal)
Load full output package into CAM tool (CAM350 or Valor). Run layer alignment, drill-to-copper overlay, and netlist compare. Resolve any discrepancies before sending to manufacturer.
Manufacturer DFM Review
Send package to FAB and assembly house. Request DFM report from FAB (Valor NPI). Review all FAB queries and DFM violations within 24 hours. Resolve or accept with written justification.
Release Package Archival
Package all outputs into versioned ZIP archive with MD5 checksum. Tag release in version control (Git). Distribute to FAB, PCBA house, and test team via secure file share — not email attachments.

Checklists

Pre-Release Checklist

DRC completed with ZERO critical violations — report saved
Design revision/version number updated in title block
All ECOs (Engineering Change Orders) incorporated and verified
Schematic cross-checked against PCB layout for all critical nets
Component footprints verified against manufacturer datasheet land patterns
Board outline verified as closed polygon — no gaps
Silkscreen: no text/lines overlapping exposed pads
All DNP components annotated in both schematic and layout

Fabrication Checklist

All copper layers exported (count = stackup layer count)
Top and bottom solder mask layers present and verified in viewer
Drill file: PTH and NPTH as separate files
Drill overlay on copper in Gerber viewer — all pads have holes
Inner plane polarity confirmed (positive vs negative)
Fabrication drawing: stackup, finish, impedance, dimensions specified
Board outline layer present (GKO or mechanical layer)
Controlled impedance traces: width matches impedance calculator result
Fiducials: minimum 3 global, correct shape, clear area around them
Gerber format: RS-274X (not RS-274D)
File naming: all files clearly labeled by layer name, no ambiguity

Assembly Checklist

BOM: every component has MPN, manufacturer, quantity, and description
BOM ref des list cross-checked against PnP ref des list — no mismatches
PnP rotation values verified for at minimum polar components (diodes, ICs, caps)
Paste mask: no apertures on vias or NPTH holes
DNP parts removed from PnP file or clearly flagged
Assembly drawing: polarity marks visible for diodes, ICs, capacitors
BOM: RoHS compliance column populated
AVL (Approved Vendor List) specified for all critical or long-lead components
Stencil thickness and aperture reduction rules communicated to assembler

Test Readiness Checklist

IPC-D-356A netlist exported and validated against CAD netlist
Test coverage calculated — ≥80% of nets have accessible test points
All power rails have at least one test point
Test point pads: minimum 1.0mm diameter (ICT) or 0.8mm (flying probe)
Test point report reviewed by test engineer
BSDL files collected from IC vendors (if boundary scan applicable)
ICT fixture design initiated (if required) — long lead time item
✅ Final Release Gate

All three checklists must be completed and signed off before the release package is sent to any external manufacturer. The release engineer must archive:

  • Completed checklist (PDF or signed document)
