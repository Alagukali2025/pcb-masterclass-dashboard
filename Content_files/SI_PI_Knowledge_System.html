SI & PI Fundamentals

Signal Integrity (SI)
The discipline of ensuring that electrical signals propagate from source to receiver without degradation — maintaining correct voltage levels, timing, and waveform shape at the target IC pin.
Power Integrity (PI)
The discipline of ensuring that the power supply voltage remains clean, stable, and within spec at every IC power pin — under all operating conditions including switching transients.
Why Both Matter
SI failures cause bit errors, timing violations, and protocol failures. PI failures inject noise into signal references, creating jitter and false switching. They are deeply coupled.

Key Concepts — Physics-Based Definitions

IMPEDANCEMAXWELL / TRANSMISSION LINE THEORY
WhatOpposition to current flow in an AC/transient sense. For PCB traces, it is the characteristic impedance Z₀ — determined by geometry and dielectric material.
WhySignals on PCB traces travel as electromagnetic waves. When the wave hits an impedance discontinuity, energy reflects. At high speed, even a trace IS a transmission line.
Z₀ = √(L/C) L = inductance/unit length | C = capacitance/unit length | Units: Ohms (Ω)
ImpactMismatched impedance → reflections → ringing → overshoot → false switching or setup/hold violations
REFLECTIONTRANSMISSION LINE THEORY (HEAVISIDE)
WhatWhen a traveling wave hits a discontinuity (impedance change, unterminated end, via, stub), a portion of the energy bounces back toward the source.
PhysicsReflection coefficient Γ = (Z_L − Z₀) / (Z_L + Z₀). Open end: Γ = +1 (voltage doubles). Short end: Γ = −1 (voltage cancels).
ImpactRinging on waveform, multiple threshold crossings, timing uncertainty, metastability in receiving flip-flop.
CROSSTALKMAXWELL — EM COUPLING
WhatEnergy coupling between adjacent traces via mutual capacitance (C_m) and mutual inductance (L_m).
TypesNEXT (Near End) — noise appears at source side.
FEXT (Far End) — noise at receiver side.
ImpactFalse switching, increased BER, jitter injection into victim net.
RETURN PATHELECTROMAGNETIC THEORY
WhatEvery signal current MUST return to its source. The return current flows directly beneath the signal trace on the reference plane — minimizing loop area.
PhysicsReturn current follows path of minimum inductance (= minimum loop area), not minimum resistance at high frequency.
ImpactBroken reference plane forces return current to detour → massive EMI loop, crosstalk spike, SI degradation.
NOISE
SourcesSwitching ICs, regulator ripple, ground bounce, coupled aggressor signals, EMI from external sources.
ImpactReduces noise margin; at sufficient amplitude, causes logic errors at IC inputs.
GROUND BOUNCEJEDEC / IBIS THEORY
WhatV = L · dI/dt transient voltage spike on GND or VDD pin of an IC due to package inductance when many outputs switch simultaneously.
ImpactFalse switching of non-switching I/Os on same package; noise injected into PDN; core voltage droop.

Signal Integrity — Core

2.1 Transmission Line Theory

Rule: When is a Trace a Transmission Line?
Condition: When trace electrical length ≥ 1/6 of the signal rise time wavelength.
Practical threshold: Trace delay (ps) ≥ T_rise (ps) / 6
Rule of thumb (FR4 ~170 ps/inch): Any trace > ~1 inch at speeds >100 MHz should be treated as a transmission line.
Source: Johnson & Graham — "High Speed Signal Propagation"

⚠ Termination — Physics-Based
Source termination: Rs in series with driver output; value = Z₀ − Z_driver. Absorbs reflections at source. No DC power dissipation.
End termination: R to VTT or AC cap at far end. Required for multidrop topologies (DDR). Dissipates DC power.
If unterminated: Open-end reflection Γ = +1. Signal doubles and bounces. At 1 GHz this is catastrophic for timing.

Reference Plane Continuity — CRITICAL RULE

🔴 Critical — Return Path Discontinuity
Rule: Every signal layer MUST have an unbroken reference plane (GND or PWR) on the adjacent layer. NEVER route a high-speed signal across a plane split.

Physics: Return current at high frequency flows directly beneath the signal trace. A split forces the return current around the gap — creating a large inductive loop.

If violated: 10–20 dB spike in insertion loss, massive EMI radiation at gap resonance frequency, inter-bus crosstalk, power noise coupling directly into signal. Boards have failed EMC testing solely due to this.
GOOD (Continuous Reference Plane): ┌────────────────────────────────────────┐ │ SIGNAL LAYER: ═══════════════════ │ ← Signal trace ├────────────────────────────────────────┤ │ GND PLANE: ████████████████████ │ ← Return current flows HERE (directly beneath) └────────────────────────────────────────┘ BAD (Plane Split Crossing): ┌────────────────────────────────────────┐ │ SIGNAL LAYER: ═══════════╪═══════ │ ← Trace crosses split ├────────────────────────────────────────┤ │ GND PLANE: ████████ │ ████ │ ← GAP in GND plane! Return current forced around └────────────────────────────────────────┘ Result: Return current detour → inductive loop → EMI + crosstalk spike

DDR Topology Note
Fly-by topology (used in DDR3/4/5): Controller → DIMM1 → DIMM2 → termination. Each DRAM sees a slightly different propagation delay. The memory controller uses write leveling to compensate DQS-CLK skew per DRAM. Incorrect routing topology breaks write leveling and causes non-functional DDR.

Power Integrity — Core

3.1 PDN — Power Distribution Network

What is PDN?
The complete network from voltage regulator (VRM) output through bulk caps, PCB planes, decoupling caps, to IC power pin. Every component adds impedance.
Target Impedance
Z_target = ΔV_allowed / ΔI_transient. The PDN impedance must stay BELOW Z_target across all operating frequencies (100 Hz – 1 GHz+).
Flat Impedance Profile
Goal: achieve PDN impedance that is flat (no peaks) below Z_target across the full frequency band. Peaks cause resonance-driven voltage spikes.

Target Impedance Formula

Z_target (Ω) = ΔV_ripple (V) / ΔI_transient (A) Example: VDD = 1.0V, ΔV = 2% = 20mV, ΔI = 2A → Z_target = 10 mΩ
⚠ FPGA / SoC Core Power
Modern FPGAs (Xilinx UltraScale+, Intel Agilex) have core VDD = 0.85V with ΔV tolerance of 3%. ΔI during configuration can exceed 10A transiently. This gives Z_target ≈ 2.5 mΩ — a demanding PDN design requiring many parallel caps and low-inductance plane pairs.
Source: FPGA vendor PDN application notes (Xilinx UG583, Intel AN 583)

🔴 CRITICAL — Placement is Paramount
The value of a decoupling capacitor is WORTHLESS if the loop inductance is too high.

Effective capacitor impedance at high frequency is dominated by ESL (series inductance), not capacitance.
Loop inductance L_loop = (via inductance + trace inductance to power pin) — this directly adds to ESL.

Target: Keep total loop inductance <100 pH for GHz-range decoupling.
Violated by: Long traces to cap pads, single via per cap, caps placed far from power pin.

3.3 Power Planes

POWER/GND PLANE PAIR CAPACITANCEELECTROMAGNETIC THEORY / IPC-2141A
WhatTwo adjacent conductive planes separated by dielectric form a distributed parallel plate capacitor. This provides inherent high-frequency decoupling with extremely low ESL.
C_plane = ε₀ × ε_r × A / d A = plane overlap area | d = dielectric thickness | Typ: 100nF per sq.inch at 4mil spacing
ImplicationUsing thin dielectric between VCC-GND pair (4 mil or less) significantly increases plane capacitance — effectively free high-frequency decoupling across the entire board. This is why power-GND pairing in stackup is a SI/PI decision.

Via Stitching

Rule
GND stitching vias must be placed around the perimeter of each power domain and along plane layer transitions.
Spacing: ≤ λ/20 at highest frequency of concern (PCIe Gen4 at 8 GHz → λ/20 ≈ 93 mils in FR4).
Why: Stitching vias maintain equipotential GND reference plane, prevent cavity resonance modes, and provide low-inductance return path at layer changes.
Source: IPC-2141A / Huray — "The Foundations of Signal Integrity"

SI vs PI Interaction

🔴 Core Concept — They Are Inseparable
SI and PI are NOT independent domains. Poor power integrity directly degrades signal integrity. This coupling is the most commonly overlooked failure mechanism in board design.
Power Noise → Signal Noise
PDN noise couples into signal reference through IC substrate. VDD ripple modulates the switching threshold of CMOS gates, creating supply-modulated jitter (periodic jitter). At 1% ripple on 1V VDD, threshold shifts by ~5–10mV — directly consuming noise margin.
Return Path → Signal Distortion
Return current flows on the reference plane. If that plane has power noise on it (e.g., a PWR plane used as reference), all signals referenced to it inherit that noise. Every mV on the reference plane = mV of noise injected into the signal.
Jitter Generation
VDD-modulated jitter is deterministic (periodic). Ground bounce from SSN generates random + periodic jitter components. High jitter consumes the timing budget of DDR/PCIe. A PDN with 50mV ripple at 500MHz can cause measurable periodic jitter in adjacent signals.

Interaction Mechanism — Detailed

MECHANISM 1: PDN Noise → IC Input Threshold Modulation
  • CMOS switching threshold V_th ≈ VDD/2 for standard logic
  • If VDD has 50mV ripple at 100MHz, V_th oscillates ±25mV at 100MHz
  • This creates supply-induced periodic jitter (SIPJ) of T_jitter = ΔV/(dV/dt of signal edge)
  • For a 100ps rise-time signal with 500mV swing: 25mV noise → ~5ps jitter at receiver
  • Source: IEEE 1057 Measurement Standards for Jitter Analysis
MECHANISM 2: Return Path Discontinuity → SI Degradation
  • Signal layer changes from Layer 2 to Layer 4 via a via
  • L2 reference = GND (Layer 1), L4 reference = GND (Layer 5)
  • Without a GND stitching via near the signal via, return current must detour around the board to Layer 5 GND
  • Loop area increases from ~0.01 mm² to possibly >10 mm² → 1000× more EMI radiation
  • Impedance discontinuity at via causes partial reflection → degraded S21 insertion loss
  • Fix: Always place GND stitching via within 50 mils of any signal via that changes reference layers

Impedance Calculation Dependencies

What Determines Trace Impedance
For a microstrip (outer layer): Z₀ depends on trace width (W), trace thickness (T), dielectric height (H), and Er (dielectric constant).
For a stripline (inner layer): Z₀ also depends on distance to both reference planes (asymmetric stripline differs from symmetric).

Practical note: Always extract final impedance from your specific fab's dielectric stack. Do NOT rely on generic Er = 4.3 for FR4 — actual Er at 1 GHz in FR4 is typically 3.8–4.2. Specify impedance target and tolerance in your PCB stackup notes; require fab to impedance test per IPC-TM-650.

Simulation & Validation

Pre-Layout Simulation

  • IBIS Models: Behavioral I/O buffer models from IC datasheets. Used for driver/receiver waveform simulation. Required for DDR, PCIe, USB SI analysis. Source: IBIS Open Forum spec v7.0
  • S-Parameters (Touchstone .s2p/.snp): Frequency-domain characterization of PCB channels, connectors, cables. Input to channel compliance simulation.
  • Topology Simulation: Validate termination, stub length, fly-by topology. Tools: HyperLynx LineSim, Ansys SIwave Schematic.
  • PDN Target Impedance: Design PDN flat impedance profile before layout. Tool: Sigrity PowerDC or manual spreadsheet using capacitor ESR/ESL values from datasheets.

Post-Layout Simulation

  • Eye Diagram: Statistical eye from bit stream simulation. Validate eye height > V_eye_min and eye width > UI_min per JEDEC/PCI-SIG spec. Failing eye = guaranteed bit errors.
  • Crosstalk Analysis: Extract NEXT/FEXT from post-layout parasitics. Add to eye diagram as worst-case noise. Identify victim-aggressor pairs.
  • PDN Simulation (Post-Layout): 3D FEM of plane + via + capacitor network. Validate impedance profile below Z_target. Check for anti-resonance peaks.
  • TDR Simulation: Time-domain reflectometry from simulation. Identifies impedance discontinuities along the signal path.

Practical Routing Workflow

1
Define Stackup First
Work with your PCB fabricator to define layer count, dielectric thickness, copper weights, and controlled impedance targets. Obtain impedance test coupons. Lock the stackup BEFORE routing begins — changing it after routing invalidates all impedance calculations. Critical
2
Import Constraints from Datasheets & Standards
Read the target IC datasheet signal routing requirements. Extract differential impedance, length matching requirements, pair skew limits, forbidden via zones, and recommended reference layer. Enter constraints into your PCB tool's constraint manager (Allegro PCB Designer, Xpedition, KiCad DRC).
3
Place Critical Components
Place DDR/FPGA/processor first. Position decoupling caps immediately adjacent to power pins (especially BGA). Orient memory ICs for minimum fly-by routing length. Place crystal/oscillator away from high-speed routing. Keep power regulators near IC power entry to minimize PDN trace inductance.
4
Route High-Speed Signals First
Route in priority: DDR CLK/DQS → DDR DQ byte lanes → PCIe/USB differential pairs → single-ended high-speed. Maintain continuous reference plane. Add return path stitching vias at every reference layer change. Apply length matching within each interface. Avoid acute angles and unnecessary vias.
5
Design PDN — Planes & Decoupling
Fill power planes for each power domain. Verify no signal crosses a power domain boundary on the plane directly below it. Place all decoupling caps (from IC datasheet BOM). Verify cap placement meets loop inductance rules. Add GND stitching vias around power domain perimeter.
6
Perform SI/PI Design Review
Run impedance DRC checks. Verify differential pair spacing and length matching. Check for reference plane discontinuities. Review via stub lengths at high-speed interfaces (back-drill if needed). Review decoupling cap placement. Confirm all return path stitching vias are present.
7
Validate with Simulation
Run post-layout SI simulation for critical interfaces (eye diagram, crosstalk). Run PDN impedance simulation — check for peaks above Z_target. Run IR drop analysis for power planes. Address any violations before releasing to fabrication. Do not skip

Common Design Mistakes

MistakeRoot CauseReal FailurePrevention
Broken Return Path Signal crosses plane split, or layer change without stitching via EMC failure (radiated emissions); inter-bus crosstalk; SI degradation; intermittent failures under thermal stress Strict rule: no high-speed signal crosses plane split. Stitching via at every layer change.
Uncontrolled Impedance Trace width not calculated for target Z₀; stackup changed after routing Reflections on DDR/PCIe → ringing → timing violations; board fails protocol compliance test Calculate trace width from stackup; lock stackup before routing; fab impedance coupon test.
Poor Decoupling Placement Decoupling caps placed far from IC, or with long traces to pads High loop inductance → cap ineffective at GHz → PDN impedance spike → VDD droop → IC malfunction Place caps within 3–5mm of IC power pin. Minimize via length and trace length to cap pads.
Excessive Vias on High-Speed Blind/through vias added for routing convenience on PCIe/USB signals Each via adds capacitive discontinuity + stub resonance. PCIe Gen4 limit: <2 vias per lane without back-drill. Minimize vias; back-drill stubs; use micro-vias for BGA fanout.
Long Stubs on DDR/PCIe Via through-hole with only partial connection — unused portion becomes stub Stub resonance at f = c/(4×L_stub×√εr) creates notch in S21 — complete signal null at resonance frequency Back-drill all vias on PCIe/USB at >3 Gbps. Use via-in-pad + plugged vias for BGA.
Ignoring PDN Impedance Placing random caps "by feel" without impedance analysis Anti-resonance peaks above Z_target → deterministic jitter in all signals; IC behavioral failures that are hard to debug Simulate PDN impedance profile. Select cap values to fill gaps and avoid anti-resonance peaks.
PWR Plane as Reference for Signals Using VDD plane as reference for signal layer instead of GND VDD plane carries switching noise. All referenced signals inherit that noise directly. PDN ripple → signal noise → bit errors. Always reference high-speed signals to GND planes, not power planes.

Debugging & Troubleshooting

SI Symptoms & Diagnosis

RINGING / OVERSHOOT
SymptomOscillating waveform after edge transition; voltage exceeds VDD or goes below GND
CauseUnterminated or under-terminated transmission line; via stub resonance; impedance discontinuity
TestTDR (Time Domain Reflectometry): incident pulse + reflection shows where impedance change occurs and its magnitude
FixAdd source or end termination; back-drill via stubs; remove pad capacitance discontinuity
TIMING VIOLATIONS
SymptomDDR training fails; intermittent data errors; CRC errors on PCIe
CauseIntra-pair skew; CLK-DQS phase error; jitter from PDN noise; signal reflection consuming UI timing budget
TestEye diagram on oscilloscope; BERT (Bit Error Rate Test); DDR training log analysis
FixLength matching; fix PDN; add/adjust termination; revise via placement

PI Symptoms & Diagnosis

VOLTAGE RIPPLE
SymptomPeriodic noise on VDD at regulator switching frequency (100kHz–2MHz) visible on oscilloscope
CauseInsufficient output capacitance; high ESR bulk caps; VRM control loop poorly compensated
TestProbe VDD at IC power pin (not VRM output) with 20MHz BW limit. Measure Vpp ripple.
FixAdd bulk MLCC (47µF–100µF); improve placement; verify VRM loop compensation
VOLTAGE DROOP / SPIKE
SymptomVDD drops below VDD_min during burst activity; spike above VDD_max during load release
CausePDN inductance too high; insufficient HF decoupling; VRM slew rate too slow for load transient
TestHigh-bandwidth probe (1GHz+) at IC power pin. Trigger on IC activity. Measure droop magnitude and duration.
FixAdd HF caps closer to IC; reduce via inductance; optimize VRM bandwidth

Checklists — Mandatory

Pre-Layout SI/PI Checklist

  • Stackup defined and impedance targets calculated with fab
  • IC datasheets reviewed for all signal routing requirements
  • DDR/PCIe/USB interface standard constraints imported into constraint manager
  • Target impedance Z_target calculated for all power domains
  • Decoupling cap BOM derived from IC datasheet and PDN simulation
  • IBIS models obtained for all critical ICs
  • Pre-layout SI topology simulation completed
  • PDN target impedance profile simulated (flat below Z_target)
  • Component placement plan approved (decoupling cap location verified)
  • Layer assignment defined: which signals on which layers and reference plane

Post-Layout SI/PI Checklist

  • All differential pairs meet impedance target ±10% (fab data or simulation)
  • Intra-pair skew within spec for all DDR/PCIe/USB pairs
  • Inter-pair length matching verified for DDR byte lanes and CLK/DQS
  • No high-speed signal crosses a power plane split
  • Return path stitching vias placed at all reference layer changes
  • Via stub length checked; back-drill specified where needed
  • All decoupling caps within 5mm of IC power pins
  • Eye diagram simulation passed per interface spec (height + width)
  • PDN post-layout impedance simulation shows no peaks above Z_target
  • IR drop simulation shows VDD within spec across all IC power pins
  • Crosstalk analysis completed; NEXT/FEXT within spec
  • Board stackup confirmed with fab — impedance coupon requested

  • Dielectric material specified (not just "FR4" — specify Dk/Df at frequency)
