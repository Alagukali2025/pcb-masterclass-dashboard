§ 01

Footprint Design Guidelines

IPC-7351 Land Pattern Calculation Methodology

IPC-7351B/C defines a mathematical framework for calculating copper land patterns based on component body dimensions, lead dimensions, and assembly class. The standard introduces three density levels (environment conditions) that govern the amount of copper extension beyond the lead:

Density Level IPC-7351 Designation Assembly Environment Typical Application
A — Most Material IPC-SM-782 Class A Low-density, large-board Prototype, hand assembly, rework-heavy boards
B — Nominal IPC-SM-782 Class B Medium-density General commercial PCBs — most common default
C — Least Material IPC-SM-782 Class C High-density (HDI) Mobile, wearable, space-constrained designs
IPC Standard
IPC-7351B Section 3.1 mandates that the nominal (B) environment be the default land pattern used unless explicitly specified by the customer or design constraint. All pad extensions in this guide assume Density Level B unless stated otherwise.

Land Pattern Calculation — Core Equations

IPC-7351B defines land dimensions through three dimensional extensions applied to component lead geometry. All variables carry tolerance bands from the component datasheet.

IPC-7351B Land Calculation Variables
Z = Lmax + 2·Jt + √(CL² + CS² + CP²)    // Outer courtyard span
G = Tmin − 2·Jh − √(CL² + CS² + CP²)    // Inner gap between pads
X = Wmax + 2·Js + √(CL² + CS² + CP²)    // Pad width

Lmax = Component lead length max    DATASHEET
Tmin = Component overall length min   DATASHEET
Wmax = Lead width max                  DATASHEET
Jt = Toe fillet goal                      IPC-7351
Jh = Heel fillet goal                     IPC-7351
Js = Side fillet goal                     IPC-7351
CL, CS, CP = Fabrication, placement, stencil tolerances   IPC-7351

Solder Joint Reliability Principles

Solder joint reliability is directly governed by three parameters derivable from footprint geometry:

  • Pad-to-lead ratio: The copper pad must extend sufficiently beyond the lead at toe, heel, and sides to form a well-defined meniscus during reflow. Insufficient extension reduces the fillet height and increases the probability of cold joints.
  • Thermal mass balance: Asymmetric pad sizes on two-pad passives (resistors, capacitors) cause tombstoning during reflow if one pad wets before the other. IPC-7351B mandates equal pad sizes for symmetric components.
  • Pad spacing: The gap between pads must be designed so that bridging probability remains below acceptable limits per IPC-A-610 Class 2 or Class 3 criteria.

Thermal Considerations

Exposed pads (EP) and thermal pads on QFN, DFN, and BGA packages require a defined thermal land pattern distinct from signal pads. IPC-7093 governs bottom-termination components (BTCs):

  • Exposed pad size: Must match datasheet specification exactly — typically 80–100% of component EP dimensions. Reduction to 80% is common to manage solder voiding.
  • Via array in thermal pad: IPC-7093A §4.2 specifies via size (0.30 mm finished drill, 0.20 mm annular ring minimum) and pitch (typically 1.0–1.2 mm) for thermal vias. Must be tented or plugged per IPC-7095C.
  • Solder paste coverage: IPC-7093A recommends 50–80% paste coverage on the exposed pad to balance thermal conductivity with void control.
§ 02

Datasheet Usage — SSOT Principle

Why the Datasheet is the Primary Authority

The component datasheet is the legal and technical specification document published by the manufacturer. It defines the actual physical geometry of the component as it will be delivered to the assembly line. IPC standards provide a calculation methodology, but that methodology is entirely dependent on dimensional inputs that can only come from the datasheet. Using any other source — prior art footprints, generic library entries, or assumptions — breaks the SSOT chain and introduces unverifiable risk.

Critical Rule
Never derive pad dimensions from a reference footprint in a generic library without cross-referencing the original datasheet. Generic library footprints may be based on incorrect, outdated, or different-density interpretations of the specification.

Critical Datasheet Sections

1. Package Outline Drawing

The package outline drawing is the dimensioned mechanical drawing of the component body and lead geometry. It contains every physical dimension required for IPC-7351 calculation. This section must be used to extract:

Dimension Symbol Usage in Footprint Source
Component body lengthL, DLmin/Lmax for G/Z calculationDATASHEET
Component body widthE, WTmin/Tmax referenceDATASHEET
Lead lengthL1, bToe extension basisDATASHEET
Lead widthb, cPad width (X) basisDATASHEET
Lead pitchePad center-to-center spacingDATASHEET
Lead coplanarityδSolder joint quality parameter — not pad geometryDATASHEET
Standoff heightA13D model alignment, paste volumeDATASHEET

2. Recommended Land Pattern

Many datasheets include a manufacturer-recommended land pattern section. This section provides the footprint dimensions as calculated by the component manufacturer, typically using a specific IPC density level. When this section exists:

  • Record the manufacturer's recommended pad dimensions as a reference value.
  • Identify which IPC density level the manufacturer used (often not stated — must be inferred by back-calculating against the IPC formula).
  • If your design requires a different density level, calculate from the package outline drawing using the appropriate IPC fillet goals — do not use the manufacturer recommendation directly.
  • If your design uses the same density level, the manufacturer recommendation can be used as a verification check against your independently calculated value.
Warning
Manufacturer recommended land patterns may target Density Level A (most material) by default to maximize solder joint reliability across all assembly environments. Always verify density level alignment with your project's assembly class requirements before adopting the manufacturer's recommendation directly.

3. Dimension Tables with Tolerances

The dimension table provides minimum, nominal, and maximum values for every controlled dimension on the package. IPC-7351B calculations require the use of min/max values only — nominal values are used exclusively for reference and 3D modeling purposes.

IPC-7351 Variable Required Datasheet Value Why Min/Max, Not Nominal
Lmax (toe calculation) Lead length maximum Worst-case toe extension — prevents under-pad at maximum lead length
Tmin (heel calculation) Component body length minimum Worst-case gap — ensures heel fillet at minimum body size
Wmax (side calculation) Lead width maximum Worst-case side extension — prevents overhang at maximum lead width

Datasheet Version Control

Datasheets are living documents. Manufacturers issue revisions (typically denoted by a letter suffix: Rev A, Rev B, etc.) that may change physical dimensions, especially when packaging processes are updated. The following must be tracked in the footprint library:

  • Datasheet revision number and date
  • Manufacturer part number the footprint was validated against
  • URL or archived PDF reference of the source document
  • Any dimension changes between revisions that impacted the footprint

Resolving Conflicts Between Datasheet and IPC

1
Document both values explicitly
Record the datasheet-recommended pad dimension and the IPC-calculated pad dimension side by side with full traceability. Never silently choose one over the other.
2
Identify the density level discrepancy
Back-calculate which density level the manufacturer used by substituting their pad dimensions into the IPC formula. This reveals whether they used Density A (conservative) versus Density B (nominal).
3
Apply project density requirement
Use the IPC-7351B calculation with the density level mandated by the project's assembly process capability and board space requirements.
4
Flag and escalate significant deviations
If IPC-calculated pad dimensions deviate more than 15% from the manufacturer recommendation, escalate for engineering review before library release. Include deviation notes in the footprint documentation field.
§ 04

Solder Joint Terminology

The three solder fillet locations — toe, heel, and side — are defined in IPC-7351B and IPC-A-610 and represent the three principal inspection points for SMD solder joint quality. Each fillet has distinct geometry, functional purpose, and failure modes.

① Toe Fillet

Location: The outermost edge of the solder joint, at the tip of the component lead — the end furthest from the component body.

Definition (IPC-7351B): The fillet formed between the end of the lead and the copper pad extension beyond the lead tip.

Functional Purpose: The toe fillet provides the primary mechanical anchor for the solder joint. It bears the majority of tensile stress during thermal cycling and mechanical loading. A well-formed toe fillet distributes stress across a larger copper-solder interface.

Reliability Impact: Insufficient toe extension results in a thin or absent toe fillet, significantly reducing joint fatigue life under thermal cycling per IPC-SM-785. IPC-A-610 Class 2 requires minimum 25% lead width toe fillet height; Class 3 requires 50%.

IPC Fillet Goal (Jt): Density B = 0.35 mm for SOIC/QFP; 0.55 mm for chip components.

② Heel Fillet

Location: The innermost edge of the solder joint, between the component lead bend and the pad copper, closest to the component body.

Definition (IPC-7351B): The fillet formed at the junction of the lead heel and the inward edge of the copper pad.

Functional Purpose: The heel fillet provides shear resistance. It prevents lead liftoff from the pad under flexural bending. The heel fillet is the primary inspection point for insufficient solder coverage beneath the lead.

Reliability Impact: A narrow or missing heel fillet is a direct indicator of insufficient solder volume or pad gap that is too large for proper wetting. IPC-A-610 Class 2 mandates minimum 25% lead thickness as heel fillet height.

IPC Fillet Goal (Jh): Density B = 0.25 mm; this is a negative value in the IPC formula, meaning the pad extends inward beneath the lead.

③ Side Fillet

Location: Along the lateral edges of the component lead, extending the solder beyond the sides of the lead width.

Definition (IPC-7351B): The fillet formed on both lateral sides of the solder joint between the lead width and the pad width extension.

Functional Purpose: The side fillet contributes to lateral mechanical stability and increases the total solder contact area, improving current-carrying capacity and reducing electromigration risk in high-current applications.

Reliability Impact: Absent side fillets indicate the pad width is equal to or narrower than the lead width, providing no lateral solder meniscus. This reduces overall joint strength. IPC-A-610 Class 2 requires side fillet presence; Class 3 requires minimum 25% of lead width.

IPC Fillet Goal (Js): Density B = 0.03–0.05 mm for gull-wing leads; chip components = 0.10 mm minimum.

Fillet Primary Stress Mode IPC-A-610 Class 2 Min IPC-A-610 Class 3 Min Consequence of Failure
Toe Tensile / thermal fatigue 25% lead width height 50% lead width height Crack initiation under thermal cycling → open circuit
Heel Shear / flexural bending 25% lead thickness height 25% lead thickness height Lead liftoff → intermittent connection
Side Lateral shear Presence required 25% lead width Reduced joint cross-section → fatigue failure under vibration

Traceability and Audit Readiness

For IPC Class 3 designs (aerospace, medical, military), the footprint library must support full forward and backward traceability. Every footprint must be linkable from:

  • Forward: Footprint → BOM Part Number → PCB Assembly Drawing → Manufactured Board
  • Backward: Manufactured Board → PCB Assembly Drawing → BOM Part Number → Footprint → IPC Calculation Record → Datasheet (archived, revision-specific)

This chain must be maintained without breaks. Any footprint that cannot be traced back to a specific, archived datasheet revision is non-conformant for Class 3 production purposes.

Solderability Considerations

  • Surface finish compatibility: HASL (Hot Air Solder Leveling) produces pad height variation of ±0.025 mm. ENIG (Electroless Nickel Immersion Gold) is flat to ±0.005 mm. Fine-pitch components ≤0.5 mm pitch require ENIG or equivalent flat finish — footprint notes must specify if HASL is prohibited.
  • Solder mask opening: Non-solder mask defined (NSMD) pads have the solder mask opening larger than the copper pad. NSMD is required for pads ≤0.5 mm pitch to prevent solder mask misregistration causing bridging. SMD (solder mask defined) is used for BGA where the solder mask controls ball diameter.
  • Paste reduction for large pads: For pads larger than 4 mm², stencil apertures must be broken into a grid to limit paste volume and control solder voiding. IPC-7525B §6 provides aperture reduction guidelines.

Assembly Tolerances

The IPC-7351B statistical tolerance stack-up uses three independent error sources combined using root-sum-square (RSS) methodology:

Tolerance Source Symbol Density B Value Description
PCB fabrication toleranceCL±0.10 mmCopper feature dimensional accuracy from fabrication
Component placement toleranceCS±0.05 mmPick-and-place machine placement accuracy
Solder paste printing toleranceCP±0.025 mmStencil printing registration accuracy
RSS Combined Tolerance
RMS = √(CL² + CS² + CP²) = √(0.10² + 0.05² + 0.025²) = ≈ 0.114 mm

§ 07

Tool-Specific Best Practices

ALTIUM DESIGNER

Library Creation Workflow

  • Use the Integrated Library (.IntLib) format for production — bundles schematic symbol, footprint, simulation model, and 3D model in a single managed unit.
  • The PCB Library Editor (.PcbLib) is the footprint authoring environment. Each footprint exists as a named component with full parameter support.
  • Use Component Templates (.PCBComponent) to standardize courtyard, assembly, and silkscreen layer rules across component families.
  • The IPC Compliant Footprint Wizard (Tools → IPC Compliant Footprint Wizard) provides a guided interface for common package types. It implements IPC-7351B formulas directly — density level selection is available. However, all wizard outputs must be verified against manual calculations.

Padstack Handling

  • SMD pads in Altium are defined at the component footprint level — pad shape (rectangular, rounded, chamfered), size, and layer assignment are set per pad.
  • Through-hole pads use the Pad Properties dialog — drill size, pad shape (circular, oval), anti-pad size (power planes), and thermal relief settings are configured here.
  • Altium does not use a separate padstack library — this simplifies setup but requires discipline to avoid pad dimension inconsistencies across footprints for similar package types.

Rule Management

  • Footprint-level DRC is performed in the PCB Library Editor using Reports → Component Rule Check.
  • Board-level clearance rules in PCB Rules and Constraints Editor (PCB → Rules) take precedence over footprint-embedded settings at layout time.
  • Use Design Rule Checks (DRC) at footprint level to catch courtyard overlaps, silk-to-pad violations, and missing layer content before library release.
CADENCE ORCAD / ALLEGRO

Library Creation Workflow

  • Cadence separates schematic symbols (OrCAD Capture, .olb) from PCB footprints (Allegro PCB Editor, .dra/.psm). These are linked at the schematic component level via the PCB Footprint property.
  • Footprints are created in Allegro PCB Symbol Editor (standalone or via Package Designer). The symbol type is Package Symbol (.psm).
  • The Padstack Editor (standalone application: padstack_editor) is a separate tool that creates reusable padstack definitions (.pad files). This is a critical distinction from Altium.
  • Cadence supports a Symbol Library (.dra) system where footprints are stored as individual files and managed through the Allegro library path configuration.

Padstack Handling

  • Padstacks are defined in the Padstack Editor as standalone .pad files, reusable across multiple footprints. This enforces standardization — the same padstack can be used in all SOT-23 footprints, for example.
  • Padstack layers: Begin Layer (copper), End Layer (copper), Default (inner layers), Soldermask Top/Bottom, Pastemask Top/Bottom, Filmmask — each configured independently.
  • Anti-pad and thermal relief shapes for power plane connectivity are defined within the padstack, not at the footprint or board level.

Rule Management

  • Constraint management is handled through the Constraint Manager (Allegro), which supports hierarchical rule sets from board-level to net-level to region-level.
  • DRC at footprint level is performed through Check → Check Symbol in the Symbol Editor.
  • Cadence supports Design Rule Check (DRC) output reports with IPC-class constraint sets, enabling automated compliance checking against IPC-2221/IPC-7351 rules when properly configured.

§ 08

Step-by-Step Footprint Creation Process

Altium Designer — SMD Component (Example: SOT-23-3)

1
Open PCB Library Editor → New Footprint
File → New → PCB Library, or open existing .PcbLib. In Component pane, right-click → Add Component. Name the footprint per naming convention: SOT-23-3_IPC-7351B_B. Enter all metadata in component parameters.
2
Place SMD Pads (Signal Pads)
Place → Pad. Set pad properties: Layer = Top, Shape = Rectangular (for standard SMD), Size from IPC calculation. For SOT-23-3: Pad size = 1.00 mm × 0.70 mm (Density B, example values — always derive from actual datasheet). Set pad number starting at Pin 1. Place all pads at exact coordinates derived from datasheet lead pitch.
3
Define Silkscreen Layer
Switch to Top Overlay (Silkscreen) layer. Draw component body outline using lines. Minimum line width: 0.12 mm. Ensure all silkscreen geometry maintains ≥0.10 mm clearance from all pad copper edges per IPC-7351B. Place Pin 1 indicator: a dot or triangle adjacent to Pad 1, within silkscreen layer, clearly visible.
4
Define Assembly Layer
Switch to Top Assembly (Mechanical 13 in older Altium; Assembly Drawing Top in newer). Draw component body outline at exact datasheet body dimensions — this is used for assembly drawings and pick-and-place programming, not for solder mask or copper. Include Pin 1 indicator on assembly layer.
5
Define Courtyard Layer
Switch to Courtyard (Mechanical 15 or dedicated Courtyard layer). Draw a rectangle encompassing the component body maximum extents plus all pads plus 0.25 mm clearance on all sides (Density B per IPC-7351B §4.1). The courtyard defines the exclusion zone for adjacent components.
6
Set Component Origin
The component origin (reference point) must be placed at the centroid of the component as specified by the IPC-7351B land pattern centroid. For symmetric components, this is the geometric center of all pads. For asymmetric packages (SOT-23-3), the centroid is at the geometric center of the pad group.
7
Align 3D Model
Tools → Manage 3D Bodies. Import STEP file from manufacturer (preferred) or IPC Compliant 3D model source. Align 3D body to component body outline exactly. Pin 1 on 3D model must coincide with Pin 1 on footprint. Verify height accuracy against datasheet overall height dimension.
8
Run DRC and Save
Reports → Component Rule Check. Verify zero violations. Save .PcbLib. Commit to version control with metadata and datasheet reference attached.

Cadence Allegro — SMD Component (Example: SOT-23-3)

1
Create Padstack (.pad) Files First
Launch Padstack Editor (standalone). New padstack: Type = SMD Regular. Layer BEGIN_LAYER: Shape = Rectangle, Width = 1.00 mm, Height = 0.70 mm. Layer SOLDERMASK_TOP: Shape = Rectangle, Width = pad + 2×0.05 mm solder mask expansion. Layer PASTEMASK_TOP: Shape = Rectangle, Width = pad − 0 mm (no reduction for standard pads). Save as SMD_1000x700.pad.
2
Open Allegro Package Designer — New Symbol
File → New → Package Symbol. Set symbol name and units (mm). Configure drawing extents. Symbol Type = Package Symbol.
3
Place Pads Using Padstack
Layout → Pins. Select padstack created in Step 1. Place pads at exact coordinates derived from IPC calculation (lead pitch, pad center positions). Assign pin numbers: Pin 1 at standard location per datasheet. Use Add → Pin command for each pad.
4
Draw Silkscreen (Package_Geometry/Silkscreen_Top)
Add → Line on class Package_Geometry, subclass Silkscreen_Top. Draw component body outline with 0.12 mm line width. Enforce ≥0.10 mm clearance from pad copper. Add Pin 1 indicator (dot or chamfer mark) adjacent to Pin 1 location.
5
Draw Assembly Outline (Package_Geometry/Assembly_Top)
Add → Line on class Package_Geometry, subclass Assembly_Top. Draw exact body outline per datasheet body dimensions. Include Pin 1 indicator matching silkscreen. This layer feeds assembly documentation and X-ray reference.
6
Draw Courtyard (Package_Keepout/All or Package_Geometry/Place_Bound_Top)
Add → Rectangle or Line on the appropriate keepout class. Define boundary at component body + pad extents + 0.25 mm on all sides. In Allegro, the placement boundary (Place_Bound_Top) is used by the DRC engine to prevent component overlap during placement.
7
Set Component Origin and Reference Designator
Setup → Change Drawing Origin to centroid location. Add Reference Designator text on Package_Geometry/Silkscreen_Top — minimum 0.8 mm height, positioned within courtyard but outside pad area. Add package value text on Assembly_Top layer.
8
Check Symbol and Export
Check → Check Symbol. Resolve all DRC violations. File → Create Symbol (.psm). Add to library path. Update DeviceFile (.dra) if full device is being created. Record metadata in symbol properties and commit to version control.

§ 09

Import / Release Checklist

This checklist must be completed in full and signed off by both the creating engineer and a designated reviewer before any footprint is released to the production library. No footprint may be used in a production PCB design without a completed checklist on record.

Category 1 — Datasheet Validation

  • Original manufacturer datasheet identified, downloaded, and archived (PDF). Revision number recorded. SSOT
  • Package outline drawing reviewed and all critical dimensions (L, T, W, lead pitch, standoff) extracted and recorded in IPC calculation worksheet. DATASHEET
  • Dimension table reviewed — min/max values used for IPC calculation (not nominal). DATASHEET
  • Manufacturer recommended land pattern (if present) reviewed and density level identified. DATASHEET
  • Part number variant confirmed — if datasheet covers multiple variants, correct variant dimensions used. DATASHEET

Category 2 — Pad Dimensions

  • IPC-7351B calculation worksheet completed with all variables (Jt, Jh, Js, CL, CS, CP) documented. IPC-7351B
  • Pad length (Z and G basis), pad width (X basis), and pad-to-pad gap verified against IPC calculation output. ±0.01mm tolerance IPC-7351B
  • Pad dimensions entered in EDA tool match calculated values within ±0.01 mm. TOOL
  • For exposed pads (QFN/DFN/BTC): EP size per datasheet; thermal via array per IPC-7093A. IPC-7093A
  • Solder mask opening verified: NSMD or SMD confirmed per design requirements; expansion value per IPC-7351B §3.6. IPC-7351B
  • Paste mask aperture verified: Area ratio ≥ 0.66; aspect ratio ≥ 1.5 per IPC-7525B. IPC-7525B

Category 3 — IPC Compliance

  • Density level (A/B/C) confirmed and matches project assembly environment specification. IPC-7351B
  • Courtyard boundary drawn at correct clearance (0.50 mm/A, 0.25 mm/B, 0.10 mm/C) per IPC-7351B §4.1. IPC-7351B
  • Pad-to-pad clearance meets IPC-2221A minimums for project class. IPC-2221A
  • For through-hole: annular ring meets IPC-2221A Table 9-2 for Class 2 or Class 3 as applicable. IPC-2221A
  • Any deviation from IPC-7351B documented with engineering justification. DEVIATION

Category 4 — Layer Content Verification

  • Silkscreen layer: Component body outline present; clearance ≥ 0.10 mm from all pad copper edges; line width ≥ 0.12 mm. IPC-7351B
  • Assembly layer: Exact component body outline at datasheet dimensions; Pin 1 indicator present. IPC-7351B
  • Courtyard layer: Closed rectangle/polygon encompassing all pads and body + required clearance. IPC-7351B
  • Reference Designator (RefDes) present on silkscreen; height ≥ 0.8 mm; positioned inside courtyard, outside pads. IPC-7351B
  • No copper, silkscreen, or paste geometry exists on incorrect layers. TOOL

Category 5 — Pin-1 and Polarity

  • Pin 1 identified on silkscreen layer with dot, chamfer, or triangle marker per IPC-7351B §3.5. IPC-7351B
  • Pin 1 marker position matches datasheet package orientation exactly. DATASHEET
  • Polarity marking present for diodes, electrolytic capacitors, and other polarized components. IPC-7351B
  • Pin numbering in EDA tool matches datasheet pin numbering — verified pad-by-pad. DATASHEET

Category 6 — 3D Model

  • 3D model (STEP) sourced from manufacturer website or IPC Compliant 3D model library. 3D
  • 3D model body aligned to assembly outline exactly — no offset or rotation error. 3D
  • 3D model Pin 1 coincides with footprint Pin 1 pad location. 3D
  • 3D model overall height verified against datasheet maximum height dimension. DATASHEET

Category 7 — DRC / ERC

  • Footprint-level DRC run in EDA tool — zero violations. TOOL
  • Courtyard overlap check: footprint courtyard does not self-intersect. TOOL
  • Silkscreen-to-pad clearance DRC passed. TOOL
  • Pad numbering continuity verified (no gaps, no duplicates). TOOL

Category 8 — Naming and Documentation

  • Footprint name follows project naming convention: [PACKAGE]_[STANDARD]_[DENSITY] (e.g., SOT-23-3_IPC-7351B_B). NAMING
  • All metadata fields populated: datasheet reference, IPC standard, density level, version, creator, reviewer. SSOT
  • Footprint version incremented if modifying existing released footprint. VERSION
  • IPC calculation worksheet archived and linked to footprint in library system. SSOT
  • Reviewer sign-off recorded (name, date, tool). APPROVAL
  • Engineering approval recorded before library release. APPROVAL
